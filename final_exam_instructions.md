## 二、考试任务

请基于教师提供的 `ershoufang_list.csv` 文件，在 Jupyter Notebook 中完成一次较完整的数据分析。Notebook 需要同时包含代码、运行结果、可视化图表和 Markdown 分析文字。

本次考试不要求建立机器学习模型，重点考查你是否能够使用课程中学习过的工具完成数据读取、数据理解、数据清洗、统计分析、可视化表达和文字解释。

建议分析内容包括但不限于：

1. 读取 CSV 数据并查看数据基本情况。
2. 说明数据字段含义、样本量、缺失值、重复值、异常值等情况。
3. 对必要字段进行清洗和类型转换，例如面积、总价、均价、建造时间、楼层、户型等。
4. 使用 `numpy`、`pandas` 完成描述性统计、分组统计、排序、筛选、透视表或交叉分析。
5. 使用 `seaborn` 和/或 `matplotlib` 绘制多张可视化图表。
6. 围绕福州二手房市场提出若干有数据支持的观察结论。
7. 使用 Markdown 对分析过程、图表含义和最终结论进行清晰说明。
8. 报告开头应包含一段摘要，简要概括本次报告做了什么分析、使用了哪些主要方法、得到了哪些主要结论。

鼓励从 CSV 中原始的文本字段开始清洗，例如从带有单位、空格、换行或混合格式的文本中提取数值信息，再生成可用于分析的数值变量。相比直接使用已经整理好的数值变量，这样更能体现数据清洗和特征整理能力，评分时会酌情加分。

数据字段一般包括：

| 字段 | 含义 |
|---|---|
| `id` | 房源编号 |
| `城市` | 城市拼音 |
| `市区` | 房源所在市区 |
| `标题` | 房源标题 |
| `户型` | 房屋户型 |
| `面积` | 房屋面积 |
| `方位` | 房屋朝向 |
| `楼层` | 楼层信息 |
| `时间` | 建造时间 |
| `所属小区` | 小区名称 |
| `所属区域` | 更具体的区域或地址信息 |
| `总价` | 房屋总价 |
| `均价` | 每平方米价格 |
| `房龄` | 爬取时使用的房龄分类 |

## 六、考试注意事项

1. 考试开始后请先确认数据文件可以正常读取。
2. 如遇到编码问题，可尝试 `encoding='utf-8-sig'`。
3. 数据中可能存在空格、换行符、单位字符、缺失值或格式不统一等问题，需要自行检查和处理。
4. 清洗数据时，应说明清洗依据，不要简单粗暴删除大量数据而不解释原因。
5. 可视化图表应有明确标题、坐标轴标签、图例或单位说明。
6. 图表数量不等于分析质量。应选择合适图表表达有意义的问题。
7. 画图时请注意图表排版，图片不要过大或过小，文字、坐标轴标签、图例应尽量清晰可读。
8. 分析结论必须来自数据结果，不得凭空下结论。
9. 可以查资料，但引用外部资料时应简要说明来源或用途。
10. 考试过程中如遇到技术问题、数据文件问题或提交问题，可以举手询问老师。
11. 不得与同学讨论思路、代码、图表和结论。
12. 不得复制他人 Notebook、代码片段或 AI 生成的整篇报告后直接提交。
13. 考试结束前请保存 Notebook，并确认所有单元格已经从头到尾顺序运行且没有报错。

## 八、可视化最低要求

Notebook 中至少应包含 4 张有效图表，建议覆盖以下类型中的若干种：

1. 类别变量的数量对比。
2. 数值变量的分布情况。
3. 不同类别下数值变量的差异。
4. 两个数值变量之间的关系。
5. 经过分组、分箱或交叉统计后的对比图。

图表不要求越多越好。每张图都应服务于一个明确的分析问题。

## 九、扣分与违规说明

出现以下情况将根据严重程度扣分：

1. Notebook 无法打开或无法运行：扣 10 至 30 分。
